Web Sites vs Web Services

Web sites have pages and forms

Web services have APIs

(Application Programming Interface)

Why to use a Web Service

  • Allows for multiple frontends.
    • JavaScript (AngularJS)
    • Android
    • iOS
  • Provide access to other developers (clients).
  • Views only access data, not concerned with presentation.
  • Greater separation of concerns.

Intro to REST

Representational State Transfer

REST is a technique to guide the design of APIs one the web. REST has these basic principles:

  • Client-Server Architecture
  • Stateless
  • Cacheable
  • Layered (works with proxies)
  • HATEOS Hypermedia As The Engine Of Application State

HTTP Verbs

REST uses a variety of HTTP verbs. Most web sites only use GET and POST, while services use many more.

  • GET - safely read data without modifying
  • HEAD - like GET, only read metadata
  • OPTIONS - find out API information about the resource
  • PUT - Create or update a resource indempotently
  • PATCH - Partially update a resource.
  • DELETE - Remove a resource.
  • POST - Anything else (usually create something).

Most Commonly Used

  • Create: POST or PUT
  • Read: GET
  • Update: PUT or PATCH
  • Delete: DELETE

Extensions to Django to make writing REST APIs in Django simple and effective.

pip install djangorestframework

Django REST Framework Essentials

APIView

An extension of Django class-based views with REST-specific features.

ModelView

Generic class for an API tied to a model.

ViewSet

A collection of views, usually list and detail views together.

Serializers

Define how to convert Python data to an interchange format (like JSON or XML).

ModelSerializer

Metaclass magic to make a serializer from a model.
